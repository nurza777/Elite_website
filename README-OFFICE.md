# Dev-сайт в офисной сети

Внутренняя версия сайта, доступная только с устройств, подключённых к офисному
WiFi. Наружу, в интернет, не выходит.

## Запуск

На офисном компьютере, который будет раздавать сайт:

1. Открыть папку с сайтом.
2. Двойной клик по **`start-office-server.bat`**.
3. В открывшемся чёрном окне появится ссылка вида `http://192.168.0.15:8080` —
   её и рассылаем команде.

Окно не закрывать: пока оно открыто — сайт работает. Свернуть можно.
Остановить — `Ctrl+C` или просто закрыть окно.

Если Python не установлен — поставить с [python.org](https://www.python.org/downloads/),
при установке отметить галочку **«Add Python to PATH»**.

## Как заходят коллеги

Открывают присланную ссылку в браузере телефона или ноутбука. Обязательно
из офисного WiFi — из дома или с мобильного интернета сайт не откроется.

Логин и пароль — те же, что и на dev-версии.

## Если не открывается

**«Не удаётся подключиться» у коллег, а на самом компьютере сайт работает**
Брандмауэр Windows блокирует входящие подключения. При первом запуске Windows
показывает окно «Разрешить доступ» — нужно нажать **«Разрешить доступ»** и
обязательно поставить галочку **«Частные сети»**. Если окно закрыли случайно,
выполнить один раз в PowerShell **от администратора**:

```
New-NetFirewallRule -DisplayName "Elite dev site" -Direction Inbound -Protocol TCP -LocalPort 8080 -Action Allow -Profile Private
```

**«Доступ только из офисной сети»**
Устройство не в офисной сети — подключиться к офисному WiFi. Гостевая сеть
часто изолирована от основной, тогда нужно подключиться к основной.

**Порт занят**
Открыть `start-office-server.bat` блокнотом и поменять `set PORT=8080` на
другой номер, например `8090`. Ссылка для команды изменится соответственно.

**Ссылка перестала работать на следующий день**
Роутер выдал компьютеру другой IP. Перезапустить `start-office-server.bat` —
в окне будет новая, актуальная ссылка. Чтобы адрес не менялся, попросить
сисадмина закрепить IP за этим компьютером в настройках роутера.

## Обновить сайт до свежей версии

```
git pull
```

Затем перезапустить `start-office-server.bat`.

## Что внутри

- `serve_lan.py` — сам сервер. Пускает только адреса локальной сети
  (`10.x`, `172.16–31.x`, `192.168.x`), остальным отвечает `403`. Пишет в окно
  лог обращений: время, IP, какая страница открыта.
- `start-office-server.bat` — запуск в один клик.
- `serve.py` — отдельный сервер для работы на своей машине (только localhost),
  для офисной сети не подходит.

## Про безопасность

Ограничение по локальной сети — на стороне сервера, обойти его из интернета
нельзя. Но вход по логину и паролю проверяется в браузере, поэтому от коллеги,
который целенаправленно полезет в исходники страницы, он не защитит. Для
внутреннего сайта в офисе этого достаточно; выкладывать эту версию в интернет
как есть — нельзя.
