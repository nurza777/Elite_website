# Dev-сайт в офисной сети

Внутренняя версия сайта, доступная только с устройств в офисном WiFi — либо,
если настроить Tailscale, ещё и с устройств, которые вы явно пригласили.
Наружу, в открытый интернет, не выходит.

## Запуск

На офисном компьютере, который будет раздавать сайт:

1. Открыть папку с сайтом.
2. Двойной клик по **`allow-firewall.bat`** — один раз, чтобы коллеги
   вообще смогли подключиться. Нажать «Да» в окне про права администратора.
3. Двойной клик по **`start-office-server.bat`**.
4. В открывшемся чёрном окне появится ссылка вида `http://192.168.0.15:8080` —
   её и рассылаем команде.

Окно не закрывать: пока оно открыто — сайт работает. Свернуть можно.
Остановить — `Ctrl+C` или просто закрыть окно.

Если Python не установлен — поставить с [python.org](https://www.python.org/downloads/),
при установке отметить галочку **«Add Python to PATH»**.

## Автозапуск при включении ПК

Чтобы сайт поднимался сам и его не приходилось запускать вручную:

Двойной клик по **`install-autostart.bat`** — один раз. После этого сайт
стартует при каждом входе в Windows на этом компьютере, окно открывается
свёрнутым. Права администратора не нужны.

Выключить автозапуск — двойной клик по **`uninstall-autostart.bat`**.

Важно: сайт стартует именно при **входе пользователя**, а не при включении
питания. Если компьютер стоит на экране блокировки и в него никто не вошёл —
сайт не работает. Чтобы он поднимался совсем без человека, попросите
сисадмина включить автоматический вход в Windows на этом ПК.

## Доступ вне офиса — Tailscale

Если нужно открывать сайт из дома или с телефона в дороге. Tailscale строит
частную сеть поверх интернета: сайт видят только те, кого вы явно пригласили,
наружу он по-прежнему не выставлен.

**На офисном ПК (том, где крутится сервер):**

1. Скачать и установить с [tailscale.com/download](https://tailscale.com/download)
2. Войти в аккаунт компании
3. Перезапустить `start-office-server.bat` — в окне появится вторая ссылка,
   раздел «Через Tailscale», вида `http://100.x.y.z:8080`

**На устройстве сотрудника:** установить Tailscale, войти в тот же аккаунт —
и открывать ссылку `http://100.x.y.z:8080`. Работает из любой сети.

Порт в брандмауэре открывать отдельно не нужно — трафик идёт через Tailscale.

### Красивый адрес вместо цифр (по желанию)

Чтобы не диктовать IP, включите в панели Tailscale **MagicDNS** — тогда адрес
станет вида `http://имя-компьютера:8080`.

Ещё лучше — выполнить на офисном ПК один раз:

```
tailscale serve --bg 8080
```

Сайт станет доступен по `https://имя-компьютера.имя-сети.ts.net` — с
настоящим HTTPS. Это не только красивее: на HTTPS браузер разрешает
`crypto.subtle`, а значит админ-панель и вход работают в полном,
«штатном» режиме.

## Как заходят коллеги

Открывают присланную ссылку в браузере телефона или ноутбука. Обязательно
из офисного WiFi — из дома или с мобильного интернета сайт не откроется.

Логин и пароль — те же, что и на dev-версии.

## Если не открывается

**«Не удаётся подключиться» у коллег, а на самом компьютере сайт работает**

Брандмауэр Windows блокирует входящие подключения. Решение — двойной клик по
**`allow-firewall.bat`**, затем нажать «Да» в окне Windows про права
администратора. Один раз.

Почему нельзя просто нажать «Разрешить доступ» в окне брандмауэра: в этом окне
по умолчанию отмечены только «Частные сети», а офисный WiFi Windows почти
всегда считает **общественной** сетью (Public) — правило для частных сетей в
этом случае не работает вообще. `allow-firewall.bat` создаёт правило сразу для
всех типов сетей, поэтому срабатывает в любом случае.

Проверить, как Windows видит вашу сеть, можно в PowerShell:

```
Get-NetConnectionProfile
```

Если в колонке `NetworkCategory` написано `Public` — это как раз тот случай.

**Не помогло даже после `allow-firewall.bat`**

Скорее всего, на роутере включена изоляция клиентов: устройства в такой сети
не видят друг друга, это частая настройка гостевых сетей. Подключиться к
основной сети вместо гостевой либо попросить сисадмина отключить изоляцию
(в настройках роутера: «AP isolation» / «Изоляция клиентов»).

**«Доступ только из офисной сети»**
Устройство не в офисной сети — подключиться к офисному WiFi. Гостевая сеть
часто изолирована от основной, тогда нужно подключиться к основной.

**Порт занят**
Открыть `start-office-server.bat` блокнотом и поменять `set PORT=8080` на
другой номер, например `8090`. Ссылка для команды изменится соответственно.

**Ссылка перестала работать на следующий день**
Роутер выдал компьютеру другой IP. Перезапустить `start-office-server.bat` —
в окне будет новая, актуальная ссылка. Чтобы адрес не менялся, попросить
сисадмина закрепить IP за этим компьютером в настройках роутера.

## Админ-панель: как правки доходят до остальных

Это главное, что нужно понимать про работу командой. Админка на офисном
сервере — по адресу `http://<ссылка>:8080/admin.html`.

**Кнопка «💾 Сохранить» — правки видит только тот, кто их сделал.**
Черновик сохраняется в браузере самого сотрудника. Ни сервер, ни коллеги
этих изменений не увидят: у каждого свой браузер — свой черновик.

**Кнопка «🚀 Опубликовать» — отправляет правки на GitHub.** Нужен токен
доступа (раздел «Публикация» в админке). Но и после публикации офисный
сервер продолжит показывать старую версию: он отдаёт файлы со своего диска,
а не с GitHub.

**Чтобы правки увидели все, на офисном ПК нужно выполнить:**

```
git pull
```

Перезапускать сервер не нужно — он читает файлы с диска при каждом запросе,
достаточно обновить страницу в браузере (F5).

Итого рабочий порядок: сотрудник правит → «Опубликовать» → на офисном ПК
`git pull` → все видят новую версию.

Если постоянно делать `git pull` руками неудобно — скажите, настроим
автоматическое обновление раз в несколько минут.

## Обновить сайт до свежей версии

```
git pull
```

Перезапускать сервер не требуется — достаточно обновить страницу.

## Что внутри

- `serve_lan.py` — сам сервер. Пускает только адреса локальной сети
  (`10.x`, `172.16–31.x`, `192.168.x`) и Tailscale (`100.64–127.x`), остальным
  отвечает `403`. Пишет в окно лог обращений: время, IP, страница и код
  ответа — строки с `<-- 404` сразу показывают, какого файла не хватает.
- `start-office-server.bat` — запуск в один клик.
- `serve.py` — отдельный сервер для работы на своей машине (только localhost),
  для офисной сети не подходит.
- `install-autostart.bat` / `uninstall-autostart.bat` — включить и выключить
  автозапуск при входе в Windows.
- `allow-firewall.bat` — разрешает входящие подключения на порт 8080 в
  брандмауэре Windows. Сам запрашивает права администратора.

## Про безопасность

Ограничение по локальной сети — на стороне сервера, обойти его из интернета
нельзя. Но вход по логину и паролю проверяется в браузере, поэтому от коллеги,
который целенаправленно полезет в исходники страницы, он не защитит. Для
внутреннего сайта в офисе этого достаточно; выкладывать эту версию в интернет
как есть — нельзя.
